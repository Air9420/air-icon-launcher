## 报告安全漏洞

如果你发现安全漏洞，**请不要公开 Issue 或 PR**。请通过以下方式私下报告：

### 报告方式

1. **GitHub Security Advisories**（推荐）
   - 访问 [Security](https://github.com/Air9420/air-icon-launcher/security/advisories/new)
   - 填写漏洞详情

2. **直接联系维护者**
   - 发送邮件至：951993674@qq.com

### 报告内容应包含

- **漏洞类型**: XSS、注入、权限提升等
- **影响范围**: 受影响的版本、功能模块
- **复现步骤**: 详细的重现步骤
- **潜在影响**: 可能造成的危害
- **修复建议**（可选）: 你建议的修复方案

## 响应时间线

1. **确认收到**: 48 小时内确认收到报告
2. **初步评估**: 7 个工作日内完成初步评估
3. **修复计划**: 根据严重程度制定修复时间表
4. **安全发布**: 修复完成后发布安全公告

## 安全模型说明

Air Icon Launcher 采用多层安全架构：

### 沙箱隔离

- 插件运行在独立的 iframe 沙箱中
- 阻止访问 localStorage、sessionStorage、indexedDB
- 阻止操作 DOM 和修改页面内容
- 阻止页面跳转和访问父窗口

### 权限控制

插件必须在 `manifest.json` 中声明所需权限：
- **低风险权限**: 自动授权
- **中风险权限**: 弹窗确认
- **高风险权限**: 需要用户输入确认文字才能授权

### 已知限制

当前版本的安全边界：
- 这是本地桌面启动器，不是浏览器型沙箱应用
- 为了支持任意本地快捷方式、可执行文件，启用了较宽的资源访问范围
- 未启用严格 CSP（Content Security Policy）
- 仅建议加载你信任的本地插件、快捷方式和资源文件

### 数据存储

- 所有用户数据存储在本地
- 默认不会将启动器数据上传到外部服务器
- 当你主动使用 AI 分类功能时，应用会将你选择的应用信息（如名称、路径、当前分类）发送到你配置的 AI 接口地址
- `AI API Key` 仅保存在应用运行时内存中，不写入 `config.json`，也不会出现在备份/导出文件中
- 配置和数据可通过备份功能导出（已排除 `AI API Key`）

## 最佳实践

作为用户，你可以：

1. **仅安装可信插件**: 只从可信来源安装插件
2. **定期备份数据**: 使用内置的备份功能定期备份
3. **保持更新**: 及时更新到最新版本以获取安全修复
4. **审查权限**: 安装插件时仔细审查请求的权限

## 安全审计

本项目欢迎安全研究人员进行负责任的安全测试。在进行任何安全测试前：

1. 请先联系维护者获得许可
2. 仅在你自己的实例上进行测试
3. 不要尝试访问其他用户的数据
4. 发现问题后按照上述流程报告

我们会在安全公告中感谢所有负责任地报告安全问题的研究者。

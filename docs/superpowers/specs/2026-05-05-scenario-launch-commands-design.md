# 场景启动命令设计（MVP）

> 日期：2026-05-05  
> 状态：已确认，待进入实现计划

---

## 1. 目标与范围

### 目标

在现有启动器能力上新增“场景启动命令”最小闭环，让用户可以通过单条命令一次性启动一组应用，形成高频触发场景。

### 本期范围（已确认）

1. 命令仅支持：`/work`、`/dev`、`/play`。
2. 执行模型仅支持“启动应用清单”，不包含 URL 与 shell 命令。
3. 场景归属入口放在“启动项右键菜单”。
4. 触发入口仅在首页搜索框，不新增独立页面和独立面板。

### 非目标（本期不做）

1. 参数化命令（如 `/dev projectA`）。
2. 场景管理页。
3. 延迟编排、步骤流、DSL、插件化命令体系。
4. 后端 Rust 配置结构改造。

---

## 2. 交互与用户流

### 场景归属配置

1. 用户在应用内启动项上右键。
2. 在菜单中使用“加入场景 -> work/dev/play”进行切换。
3. 支持“移出全部场景”（仅在该项已加入任一场景时显示）。

### 场景命令执行

1. 用户在首页呼出后输入 `/work`、`/dev` 或 `/play`。
2. 搜索结果顶部出现“场景启动”分组。
3. 用户按 `Enter`（或快捷键）执行该场景批量启动。
4. 启动后提示执行结果（成功数/跳过数）。

---

## 3. 数据模型与持久化

### 存储位置

放在 `launcherStore`（前端 Pinia 持久化），不改 Rust `launcher_data` 结构。

### 数据结构

```ts
type ScenarioKey = "work" | "dev" | "play";

type ScenarioItemIds = {
  work: string[];
  dev: string[];
  play: string[];
};
```

### 规则

1. 场景内按 `itemId` 去重。
2. 允许同一启动项属于多个场景。
3. 启动时按 `scenarioItemIds[scenario]` 顺序执行。
4. 删除启动项后自动清理场景中的失效 `itemId`。

### 持久化字段

在 `launcherStore` 的 `persist` 白名单新增 `scenarioItemIds`。

---

## 4. 菜单模型与动作

### 菜单扩展点

在 `IconItem + category=true` 上新增内置菜单分组：

1. 分组标题：`加入场景`
2. 子项：`work`、`dev`、`play`
3. 模式：`checkbox`
4. `checked`：由当前 `itemId` 是否在对应场景中决定
5. 行为：toggle（已加入则移出，未加入则加入）

可选动作：

1. `移出全部场景`（仅当当前项在任一场景中时可见）

---

## 5. 搜索命令识别与结果渲染

### 识别规则

1. 当 `searchKeyword.trim()` 以 `/` 开头时进入命令匹配。
2. 首版只做精确命中：`/work`、`/dev`、`/play`（兼容尾部空格）。

### 结果分组

1. 在 `SearchResults` 中新增“场景启动”分组。
2. 分组建议位于所有搜索结果顶部，减少击键成本。
3. 每条结果显示：
   - 标题：`启动场景 work|dev|play`
   - 副标题：`将启动 N 个应用`
   - 匹配标签：`精确`

### 索引约束

命令分组必须并入统一索引计算，保证显示与可选行一致：

1. `totalSearchItemCount`
2. 上下键移动
3. `Ctrl+1..0`
4. 回车执行

---

## 6. 执行链路

### 启动流程

1. 根据场景拿到 `scenarioItemIds`。
2. 通过现有 `launcherItemsByCategoryId` 反查仍存在的启动项。
3. 逐项复用现有启动能力（与普通启动项保持一致的冷却、统计和最近使用行为）。
4. 缺失项或无效项跳过并计数。

### 结果反馈

执行后统一提示：

`已启动 X 项，跳过 Y 项`

---

## 7. 验收标准

1. 右键可加入/移出 `work`、`dev`、`play`。
2. 重启应用后场景归属保持不丢失。
3. 输入 `/work`、`/dev`、`/play` 可出现并执行场景命令。
4. 命令分组支持 `Ctrl+1..0`、上下键、回车。
5. 批量启动后最近使用、冷却态、统计更新正确。

---

## 8. 回归清单

1. 普通搜索分组（应用/剪贴板/最近文件/扫描回退）行为不回退。
2. 搜索索引无漂移，不出现“显示一项选中另一项”。
3. 删除启动项后场景失效引用自动清理，不抛异常。
4. 外部项相关菜单（屏蔽、转为应用内）保持原行为。
5. 首页与分类页现有键盘轮转（Tab/ESC）不被破坏。

---
